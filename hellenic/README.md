# Hellenic Protocol Stack

Each protocol defines a workflow for a specific task type. Choose based on what you're doing.

| Protocol | Use When | Input | Output | Duration |
|---|---|---|---|---|
| **BETA** | Auditing current state, discovering risks, grounding in reality | Task description | Risk report, grounding artifacts | 30-60 min |
| **ALPHA** | Planning architecture, decomposing complex tasks, designing solutions | Problem statement | Architecture plan, design doc, task breakdown | 30-90 min |
| **GAMMA** | Executing precise, scoped work with manifest and evidence | Detailed spec + manifest | Completed work, receipt, evidence | Variable |
| **DELTA** | Resolving conflicts, reconciling stale claims, handling disputes | Conflicting claims + evidence | Reconciliation decision, claim classification | 30-120 min |
| **THETA** | Post-execution validation, verifying artifacts, ensuring quality | Artifacts + acceptance criteria | Validation report, issues or sign-off | 30-60 min |
| **GAMMA-BURST** | Doing a one-time major refactor or reconstruction | Current state + target state | Completed reconstruction, final report | 4-24 hours |

## Quick Guide

1. **Discovering the problem?** → **BETA** (audit and ground first)
2. **Planning the solution?** → **ALPHA** (design and decompose)
3. **Implementing the solution?** → **GAMMA** (exact execution)
4. **Fixing conflicts?** → **DELTA** (reconciliation)
5. **Verifying the result?** → **THETA** (validation)
6. **Rebuilding a major system?** → **GAMMA-BURST** (full reconstruction)

You typically use **BETA → ALPHA → GAMMA → THETA** in sequence.

Use **DELTA** anytime claims conflict.

Use **GAMMA-BURST** sparingly for one-time massive refactors.
