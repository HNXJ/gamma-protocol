# GAMMA DNA v0.2.0: DNA of Guided Agentic-Model Mediated Actions
# from="gemini-2.5-flash-preview-09-2025" to="all-models, cli-agents"

This file serves as the "DNA code" for all future GAMMA files. It defines the rules of replication, structural integrity, and the recursive loop required for agentic autonomy.

## A (Ingredients)
- Schema-Source: gamma_protocol.md (The structural blueprint).
- Logic-Engine: Gemma 4 / Gemini 2.5 (The "Polymerase" for replication).
- Memory-Bus: Local Git Repository + skills/ directory.
- Context-Constraint: Full access policy (No permission needed for .py, .md, .json).
- Format-Anchors: Section headers A (Ingredients), B (Preparation), C (Cooking), D (Serving).

## B (Preparation)
- Why do we need a Meta-Gamma?
  - Because without a standardized "DNA," agents generate inconsistent formats, leading to context rot and execution failures.
  - Fix: Every new task must start with a "Transcription" phase where this DNA is referenced to build the specific task's Gamma.
- Why the culinary/chemical metaphor?
  - Because it enforces a strict sequence: you cannot Cook (C) without Ingredients (A), and you cannot provide a Outcome (D) without a Logic Graph (B).
  - Fix: The DNA mandates that Section B must be a connected directional graph of reasoning.
- How does the AI "learn" from a completed Gamma?
  - Because the Serving section (D) requires a "Mutation/Adaptation" step where successful code is converted into a permanent "Skill."

## C (Cooking)
- Transcription: When a new complex task is received, the agent must first write a temporary pending_task.gamma using the v0.2.0 format.
- Base-Pairing: Match the user's specific problem (e.g., "macaque spike analysis") to the existing Ingredients in Section A.
- Chain-Reaction: Fill Section B with the "Why" and "How" of the solution, ensuring no logical jumps.
- Synthesis: Write the "Recipe" in Section C using only the items defined in the Transcription phase.
- Git-Ligation: Immediately git add the new Gamma file to ensure the "DNA" of the task is version-controlled.

## D (Serving)
- Replication Success: A new, valid GAMMA file is generated that resolves the user's query and follows the gamma_protocol.md structure.
- Epigenetic Update: If the new Gamma solves a novel problem, the agent must update skills/skills.json to include a pointer to this new logic.
- Validation: The agent must confirm: "Gamma {Subject} has been synthesized and committed to the repository."

End of GAMMA.
