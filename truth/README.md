# Truth Plane: Validation and Commitment

## Truth Hierarchy

**Proposal** → **Evidence** → **Receipt** → **Canon**

- Proposal: Any claim not yet validated
- Evidence: Execution output supporting a claim
- Receipt: Validated evidence with judge sign-off
- Canon: Committed truth in gamma backend

## Requirements for Truth Commit

1. Receipt is generated
2. Receipt includes validation checks
3. Validation checks pass
4. Judge signs or backend gate authorizes
5. Commit is irreversible
