# Truth/Receipt Doctrine

Truth requires receipts. Without a receipt, a claim is not truth.

Receipt fields:
- `receipt_id`
- `timestamp`
- `source run`
- `mission/event`
- `input state reference/hash`
- `output state reference/hash`
- `adapter/gate`
- `validation checks`
- `rejected invalids`
- `NaN/inf checks if numerical`
- `commit decision`
- `judge/signer if applicable`
- `provenance`
- `reproducibility instructions`
