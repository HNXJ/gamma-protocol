# Missions: The Smallest Meaningful Work Unit

## What is a Mission

A **mission** is the smallest scoped, completable unit of work in Gamma.

- Has explicit inputs, outputs, validation criteria
- Lives on the Control plane
- Assigned to a specific agent or agent group
- Has a receipt requirement
- Has rejection criteria

## Mission Template

Use `templates/mission_template.md` to create new missions.

## Continuous Growth Mission

See `missions/continuous_growth.md` for the prime ongoing mission of neural substrate expansion.

## Mission Lifecycle

1. **Proposal** (Control) - Mission is proposed
2. **Planning** (ALPHA) - Architecture and decomposition
3. **Execution** (GAMMA) - Agent executes
4. **Validation** (THETA) - Validator checks
5. **Resolution** (Truth plane if receipt-required) - Truth is updated or mission is rejected

## Mission Status

- **Backlog** - Not yet active
- **In Progress** - Active, agent is working
- **Review** - Completed, awaiting validation
- **Done** - Completed and validated
- **Blocked** - Cannot proceed, blocker documented
